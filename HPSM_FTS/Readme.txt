Предустановки (или будет дистрибутив включающий все, кроме офиса)
1. MS Office начиная с 10 версии. Лучше 16, быстрее работает.
2. .NET framework 4.6.1
3. Dll для связи офиса и .NET framework. Они разные для разных
  офисов. Нужно или ставить поодержку из интернета или поставлять
  набор Dll для каждой версии офиса. Мы подготовим нужные наборы.
  Прежде всего для 16 и 13 версии.

Файлы, необходимые для запуска утилиты (или будет дистрибутив 
						включающий все, кроме офиса)
1. HPSM_FTS.exe - сама утилита
2. NLog.config - конфиг системы логгирования NLog
3. NLog.dll, NLog.xml - реализация NLog
4. Выгрузка контактов.xlsx - контакты из HPSM, 
   файл должен находится в папке программы. Формат меняться
   не должен. Количество контактов может менятся.
5. Сетевые имена.txt - IP и сетевые имена АРМ.
   файл должен находится в папке программы. Формат меняться
   не должен. Количество имен может менятся.  
6. Файл выгрузки. Выбирается при запуске обработки.
   Может располагаться в любой папке и иметь любое название.
   Пример: выгрузка 18.02.2019.xlsx
7. Необходимые dll для запуска утилиты. Список будет уточнен или
   просто подготовим дистрибутив и туда все включим

Результат работы
1. Excel файл с 3 листами: Акты выполненных работ, открытозакрыто и Общая.
2. Журнал работы, который воводится в окно программы и пишется на диск
   Пример: log_20022019 110210.txt

Детали журнала
1. Основные операции - запуск, загрузка данных, сохранение данных, завершение
   работы
2. Если время, потраченное на инцидент не укладывается в норматив, 
   утилита подгоняет время открытия и закрытия чтобы норматив выполнялся.
   Это отображается в журнале.
   Пример: Инцидент ИНЦ000240986: превышен норматив времени закрытия
3. Если контакты пользователся не найдены в файле Выгрузка контактов.xlsx, 
   происходит запись в журнал
   Пример: Внимание!: Контакт для пользователя Бойко Светлана Владимировна 
      (BoykoSvV@ca.eais.customs.ru) не найден
4. Если IP и сетевое имя АРМ пользователся не найдено в файле Сетевые имена.txt,
   пишется в журнал соответствующее сообщение
   Пример: Внимание!: Не найдено сетевое устройство для пользователя Гарбук Ирина 
   Анатольевна (GarbukIA@ca.eais.customs.ru)
5. Если инцидент не закрыт в файле выгрузки данных из HPSM, пишется сообщение
   Пример: Инцидент ИНЦ000243809 не закрыт
6. При возникновении ошибки она выведется в лог. Ошибку можно скопировать и 
   оправить разработчикам.

Дополнительная информация
1. При любых изменениях структуры колонок входящих файлов - выгрузки данных, контактных
   данных, сетевых имен нужно изменять программу. 
2. Если нужно подправить данные, например, колонку Вид работ, следует модифицировать 
   исходный файл выгрузки данных из HPSM и заново обработать его.
3. По умолчанию обрабатываются только 2 рабочие группы из выгрузки - АСВДТО и 
   ОДСИПЕАИСТОГПДС. Если нужно обработать все остальные рабочие группы - нужно
   снять галку с чекбокса.
